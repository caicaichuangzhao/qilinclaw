# HEARTBEAT.md - Agent主动跟进事项

这个文件用于Agent的心跳机制，Agent会定期检查这个文件中的内容，并主动跟进需要处理的事项。

## 使用方法

1. 在这个文件中添加需要Agent主动跟进的事项
2. Agent会在每次心跳时检查这个文件
3. 如果发现有需要跟进的事项，Agent会主动向用户发送消息

## 示例格式

### 待办事项
- [ ] 每天早上9点检查服务器状态
- [ ] 每小时检查是否有新邮件
- [ ] 每周一检查项目进度

### 提醒事项
- [提醒] 每天下午5点提醒用户休息
- [提醒] 每周五提醒用户提交周报

### 监控事项
- [监控] 检查API服务是否正常运行
- [监控] 检查数据库连接状态

## 注意事项

- Agent会在心跳时自动检查这个文件
- 心跳间隔可以在Agent配置中设置
- 如果没有需要跟进的事项，Agent不会主动发送消息
